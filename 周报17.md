lambda executor, target: (target.__del__(), setattr(target, 'alive', False), '处决成功')

```

```

```

| 管道
> 重定向
{} 花括号展开
$() 命令替换
&& || 逻辑连接

`exec(单字符串)` 就像直接喊操作系统干活，操作系统不认识 `|` `{}` 这些符号；

这就造成了前端（Node.js 代理）和后端（Rust 服务器）对同一个请求 body 长度的理解不一致：

• Node.js 看到 Transfer-Encoding: chunked，按分块规则解析，读完一个完整的分块请求

• Rust 看到同样的数据，直接找 Content-Length，找不到就当 body 是 0 字节



```
请求走私利用的就是代理和后端「对请求边界理解不一致」这个漏洞。攻击者可以精心构造一个请求，让代理认为是一个请求，而后端却认为是两个请求。
```

本题是 **TE.CL（Transfer-Encoding / Content-Length）** 类型的走私：

• TE = 代理优先使用 Transfer-Encoding: chunked 解析请求

CL = 后端优先使用 （或找不到就默认0）
