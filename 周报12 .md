#### 原理理解：

1. Flask 为什么能把 /flag 这种文本文件当模板？

Flask 使用的是 Jinja2 模板引擎。
而 Jinja2 的设计是这样的：

只要是一个存在的文件，并且能被打开读取，它就能作为模板。

模板 ≠ 只有 HTML
模板可以是：

HTML

文本文件

JSON

YAML

ANY FILE（只要能读取）

模板系统最基本的功能就是：

读取这个文件 → 按行解析 → 按 Jinja 语法执行（如果文件里没有语法，就原样输出）

✔ 如果文件里什么 Jinja 语法都没有，比如：

FLAG{xxxx}

那么渲染结果就是：

FLAG{xxxx}

🧨 所以，只要文件能被读取，它就能成为一个“模板”，渲染时内容原封不动输出。

这就是为什么：

render_template("flag")

能够输出 flag 文件内容。

🔥 2. 那模板文件在哪里找？

正常 Flask 会从：

/your_app/templates/

找模板文件，因为 Flask 初始化时有一个：

app.jinja_loader.searchpath = ['/path/to/templates']

比如一个 Flask 项目的正常 searchpath：

['/app/templates']

🔥 3. 你通过 pydash 任意属性写 → 把 searchpath 改成了 '/'

你执行的：

GET /operate?username=app&password=jinja_loader.searchpath&confirm_password=['/']

会被执行成：

app.jinja_loader.searchpath = ['/']

这意味着：

从此以后，Flask 渲染模板时会从系统根目录 / 搜索模板文件。

于是：

render_template("flag")

搜索顺序变成：

/flag ← 如果存在，直接打开 /templates/flag (不再使用)

🔥 4. 于是 /flag 能被当作模板读取输出

流程如下：

用户访问

GET /impression?point=flag

Flask 调用

render_template("flag")

Jinja2 搜索 template：

检查 /flag 是否存在

如果 /flag 存在（例如题目把 flag 放在这个位置）：

打开 /flag

读文件内容

里面没有任何 Jinja 语法，所以直接 原样输出

最终页面返回：

FLAG{xxxxxxx}

🧠 一句话总结

只要你能让 Flask 认为某个路径是“模板所在目录”，
你就能让 Jinja2 去加载任意文件，作为模板渲染，从而读取其中的内容。

这是典型的：

模板搜索路径劫持 → 任意文件读取（LFI）


`render_template` 的核心使命是**加载并渲染前端模板文件**，将后端 Python 数据与前端模板进行整合，最终生成完整的 HTML 页面（或其他格式的响应内容）并返回给客户端浏览器，实现动态网页的展示。

`render_template` 会自动在 Flask 项目的 ** 默认模板目录（根目录下的 `templates` 文件夹）** 中查找指定名称的模板文件（常见为 `.html` 格式，也支持 Jinja2 模板格式）。



- 无需填写模板文件的完整路径，只需传入文件名（如 `index.html`）即可加载；
- 若需加载子目录下的模板，可使用相对路径（如 `admin/dashboard.html`，对应 `templates/admin/dashboard.html`）。

配置端口与关联文件（核心步骤）

这一步就是回答你问的“调配置文件”和“关联端口”。

Nginx 的配置文件通常在 `/etc/nginx/sites-available/` 目录下。

1. **编辑默认配置文件**

   sudo nano /etc/nginx/sites-available/default

2、**找到 server 块（这就是配置文件里的关键部分）**
你需要找到类似下面这段代码：

```
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    # 这里是根目录设置
    root /var/www/html;
    index index.html index.htm index.nginx-debian.html;

    server_name _;
    ...
}
```

配置解析

- `listen 80;`

  ：这就是

  端口配置

  。它告诉 Nginx 监听 80 端口（HTTP 默认端口）。

  - *如果你想用别的端口（比如 8080），就改成 listen 8080;。但记得去 VPS 控制台安全组放行 8080 端口。*

- **root /var/www/html;**：这就是**目录关联**。它告诉 Nginx：“你要展示的文件，在 `/var/www/html` 这个文件夹里”。

- **index index.html ...**：这就是**首页设置**。它告诉 Nginx：“当有人访问时，优先找 `index.html` 这个文件”。因为我们刚才已经在这个目录下创建了 `index.html`，所以它会被自动识别为首页。

**php://filter/[过滤模式]/[过滤链]/resource=[目标资源]**

**string.strip_tags 过滤器**（关键处理环节）

string.strip_tags 移除数据流中的所有 HTML 标签和 PHP 标签（包括<?php ?>、<? ?>等标记）

此处该过滤器的核心用途：精准剔除写入内容中的<?php exit();``?>

**完整原始内容：**

**<?php exit();php://filter/write=string.strip_tags/?>php_value auto_prepend_file /flag\n#/resource=.htaccess**



经过string.strip_tags过滤后：

* <?php exit();`（PHP 开放标签 + 内容）被完整移除

- `?>`（PHP 闭合标签）被完整移除
- 剩余有效内容：`php_value auto_prepend_file /flag\n#`（`\n`是换行符，`#`是注释符）



resource=.htaccess`（目标写入文件）

`php://filter`的`resource`参数指定了最终的写入目标文件为`.htaccess`—— 这是 Apache 服务器的分布式配置文件，支持通过特定指令配置 PHP 运行参数（仅在 Apache+PHP 模块模式下生效）。

 

`.htaccess` 配置指令：`php_value auto_prepend_file /flag`（攻击核心目的）

```
php_value auto_prepend_file [文件路径]
```



**在每一个 PHP 脚本执行之前，服务器会自动预先包含（引入）指定路径的文件内容，相当于在所有 PHP 脚本开头自动添加include('/flag');**。

由于`/flag`通常是 CTF 场景或服务器中存放敏感信息 / 密钥的文件，该指令会让后续任意 PHP 脚本执行时，先读取`/flag`的内容并输出（或纳入执行流程），从而实现敏感文件读取的目的。



注释符 `#` 的作用

变量值中的`#`是 Apache 配置文件的注释符，其作用是**将后面的/resource=.htaccess注释掉**，避免该伪协议相关的无效内容被写入`.htaccess`，防止干扰配置文件的解析（确保只有`php_value auto_prepend_file /flag`生效）。

`import pickle`：Python 内置的**序列化 / 反序列化模块**，负责将 Python 对象与字节流相互转换（`dumps()` 序列化：对象→字节流；`loads()` 反序列化：字节流→对象），这段代码的核心漏洞就源于该模块的特性。

`import os`：Python 提供的**操作系统交互模块**，这里用到 `os.system()` 函数，用于执行操作系统层面的命令（如 Linux 的`curl`、`ls`，Windows 的`dir`等）。



`__reduce__`是 pickle 模块的专用魔法方法

当`pickle.dumps()`序列化某个对象时，如果该对象的类定义了`__reduce__`方法，pickle 会优先采用该方法的返回值作为序列化内容，而非默认序列化对象的属性。

更关键的是：当后续对该序列化字节流执行`pickle.loads()`反序列化时，`__reduce__`方法返回的内容会被**自动执行**，用于重建对象（这也是 pickle 反序列化漏洞的根源）。

该方法返回元组 `(eval, (超长恶意Python字符串,))`，遵循 pickle 对`__reduce__`返回值的解析规则：

第一个元素`eval`：Python 内置的**字符串代码执行函数**（可将字符串作为合法 Python 代码执行），相比上一段的`os.system`，`eval`更隐蔽，可执行复杂嵌套代码，更难被简单安全规则检测；

第二个元素：一个元组，存放`eval`的执行参数 —— 即那段超长的嵌套 Python 字符串，这是「植入后门」的核心逻辑。



这段字符串的核心目标：**找到 Flask 应用的app实例，向其前置钩子函数中注册一个恶意命令执行函数，实现持久化后门**。

#### Flask内存马：

内存马（memory-based webshell）因“无文件落地”“难以溯源”的特性成为高级攻击的常用手段

Flask作为轻量级Python Web框架，其灵活的路由机制和动态代码执行特性，为内存马注入提供了天然条件。

Flask内存马的核心是“在应用运行时，通过动态注入恶意代码修改内存中的路由表，实现对请求的恶意响应”，整个过程不涉及磁盘文件写入，仅在内存中完成后门部署。

**路由映射在应用运行时可动态修改**。即使应用已启动，通过`app.add_url_rule()`或直接操作`url_map`，仍能添加新路由——这是Flask内存马的核心实现基础。

传统Webshell依赖写入磁盘文件（如shell.py），而Flask内存马通过以下方式实现无文件驻留：

利用应用自身的代码执行漏洞（如SSTI、eval/exec滥用），在内存中动态注册恶意路由；
恶意路由的视图函数仅存在于内存中，不落地为文件；
只要Flask应用不重启（进程不终止），恶意路由就会持续生效，重启后因内存释放而失效（但可通过持久化手段重注入）

危险函数 merge：

这段代码实现了一个字典合并的功能。函数`merge`接受两个参数`src`和`dst`，分别表示源字典和目标字典。

代码通过迭代源字典的键值对，对于每个键值对进行如下操作：

1. 首先判断目标字典`dst`是否存在`__getitem__`属性，即是否可通过索引或key来获取值。这是为了处理`dst`既可以是字典类型，也可以是对象类型的情况。
2. 如果目标字典`dst`存在键`k`，并且其对应的值不为空且值的类型为字典，则递归调用`merge`函数，将源字典`v`和目标字典`dst[k]`作为参数进行合并操作。
3. 如果目标字典`dst`通过`__getitem__`方法获取键`k`时返回的值不为空且值的类型为字典，则递归调用`merge`函数，将源字典`v`和目标字典通过`getattr`方法获取到的属性`k`作为参数进行合并操作。
4. 如果以上条件都不满足，则将源字典的键值对直接设置到目标字典中，可以通过`dst[k] = v`或`setattr(dst, k, v)`来实现。
5. *hasattr* 函数用于检查对象是否具有指定的属性或方法。它接受两个参数：对象和属性名，并返回一个布尔值

这样，整个过程会将源字典中的所有键值对逐个合并到目标字典或对象中，最终实现了两个字典的合并。

raw 即 “原始”，表示 HackBar 会完全保留你输入的 POST 数据内容，不做任何自动的 URL 编码处理，直接将原始字符串作为请求体发送。
与之对比：
普通的 application/x-www-form-urlencoded：HackBar 会自动对数据中的空格、特殊字符（如 &、=、#、中文） 进行 URL 编码（比如空格→+，中文→%XX，&→%26）。
application/x-www-form-urlencoded (raw)：不做任何编码操作，你输入什么，请求体里就发送什么。
```

```
3、application/json 的核心作用
它的作用是声明请求体的内容是标准的 JSON 字符串，并通过 Content-Type: application/json 请求头告知服务器：请使用 JSON 解析器来处理请求体数据。
与 application/x-www-form-urlencoded 的键值对字符串、multipart/form-data 的边界符分段格式不同，它直接传输结构化的 JSON 数据；
支持复杂数据类型（嵌套对象、数组），无需额外编码转换，是现代接口传输数据的首选方式；
数据不会被 URL 编码，完全保留 JSON 原生格式（比如引号、大括号、中括号）

### 跨站脚本攻击叫CSS还是XSS

CSSLeak即通过CSS注⼊实现XSLeak,⼀个常⻅的⽅法是利⽤CSS选择器匹配指定标签的某个属性
的内容

属性选择器的核心形态与常见类型

类型语法格式含义示例（匹配你的 meta 标签）

1. 存在性选择器`[attr]`选中**拥有 attr 属性**的所有元素（不管属性值是什么）`meta[name]` → 选中所有带 name 属性的 meta 标签
2. 精确值选择器`[attr="value"]`选中 attr 属性值**完全等于**value 的元素（区分大小写）`meta[name="secret"]` → 精准选中你的 meta 标签
3. 包含值选择器`[attr*="value"]`选中 attr 属性值**包含 value 字符串**的元素（任意位置）`meta[content*="ww"]` → 匹配 content 含 "ww" 的 meta
4. 开头匹配选择器`[attr^="value"]`选中 attr 属性值**以 value 开头**的元素`meta[content^="w"]` → 匹配 content 以 "w" 开头的 meta
5. 结尾匹配选择器`[attr$="value"]`选中 attr 属性值**以 value 结尾**的元素`meta[content$="w"]` → 匹配 content 以 "w" 结尾的 meta
6. 空格分隔匹配`[attr~="value"]`选中 attr 属性值是**空格分隔的列表**，且其中包含 value 的元素`div[class~="box"]` → 选中 class 含 "box" 的 div（如 class="box red"）
7.  连字符分隔匹配`[attr="value"]`选中 attr 属性值**以 value 开头且后跟连字符**，或完全等于 value 的元素`p[lang="en"]`→ 匹配 lang="en"或 lang="en-US" 的 p 标签

明确`app.run()`中`host`和`port`的不同作用：

1. **host='0.0.0.0'：控制 “哪些网卡能访问服务”**

   这是指 Flask 服务会监听服务器的**所有网卡**

   举例：你的服务器有公网 IP`8.148.230.29``172.19.24.189``host='0.0.0.0'``host='127.0.0.1'`

2. **port=8080：控制 “服务绑定的端口”**

   Flask 服务只会**绑定并监听 8080 这一个端口**`http://你的IP:8080/xxx``/next``/leak``/exp.html`

总结：

- 你的 Flask 服务**仅在 8080 端口提供这些路由**，其他端口无法访问这些接口；
- `host='0.0.0.0'`只是让 8080 端口的服务能被服务器的所有网卡（公网、内网）访问，和 “端口范围” 无关。

### **JS获取DOM元素的8种核心方法**（XSS漏洞利用关键）

1. 通过ID获取

   （

```
   getElementById
   ```

   - 用法：`document.getElementById('id')`

2. 通过name属性获取

   （

   ```
   getElementsByName
   ```

   ）

   - 用法：`document.getElementsByName('name')`
   - 特点：返回**类数组集合**，无匹配返回空数组
   - 限制：主要适用于表单元素（`<form>`、`<input>`等）

3. 通过标签名获取

   （

   ```
   getElementsByTagName
   ```

   ）

   - 用法：`document.getElementsByTagName('div')`或`element.getElementsByTagName('div')`
   - 特点：返回**动态集合**，无匹配返回空数组
   - 优势：**兼容性好**，支持所有浏览器

4. 通过类名获取

   （

   ```
   getElementsByClassName
   ```

   ）

   - 用法：`document.getElementsByClassName('class')`或`element.getElementsByClassName('class')`
   - 特点：返回**动态集合**，无匹配返回空数组
   - 限制：**IE8及以下不兼容**

   （

   ```
   querySelector
   ```

   ）

   - 用法：`document.querySelector('div.header')`

   ```
